LLMOフレームワークは、AIシステムがコンテンツを発見し、理解し、正確に引用できるかどうかを決定する5つのコアコンポーネントを定義する。

## 5つのコンポーネント

### 1. ナレッジクラリティ
コンテンツは、AIが正確に理解・要約できるほど明確か？

- 平易で曖昧さのない表現を使う
- 重要な用語を明示的に定義する
- 構造化された事実（誰が、何を、いつ、どこで）を提供する
- 説明なしの専門用語を避ける

### 2. 構造化フォーマット
コンテンツは機械が消費しやすい構造になっているか？

- セマンティックHTMLとMarkdownを使う
- JSON-LD構造化データを実装する
- AI向けコンテンツとしてllms.txtを提供する
- コンテンツを階層的に整理する

### 3. 検索シグナル
AIシステムは必要なときにコンテンツを見つけられるか？

- クロール可能性を確保する（robots.txt、sitemap.xml）
- 機械可読なエンドポイントを提供する（/ai/、.mdファイル）
- llms.txt標準を実装する
- 可能な場合はAPI経由でコンテンツを提供する

### 4. 権威性シグナル
コンテンツは専門性と信頼性を示しているか？

- 検証可能な資格情報を持つ著者の帰属表示
- クロスプラットフォームでのプレゼンス（GitHub、LinkedIn、出版物）
- すべてのプラットフォームで一貫した情報
- 引用に基づくエビデンスベースの主張

### 5. 引用シグナル
コンテンツにはAIが検証できる参考文献が含まれているか？

- 一次情報源へのリンク
- 公開日の記載
- バージョン情報の提供
- 学術論文や公式ドキュメントの参照

## スコアリング

各コンポーネントは0-3のスケールで採点できる:

| スコア | レベル | 説明 |
|-------|-------|------|
| 0 | なし | コンポーネントが対応されていない |
| 1 | 基本 | 最小限の実装 |
| 2 | 良好 | 改善の余地はあるが堅実な実装 |
| 3 | 優秀 | ベストプラクティスに沿った実装 |

**最大スコア: 15点**（5コンポーネント x 各3点）

## 自己採点チェックリスト

自サイトを各コンポーネントで採点する。自信を持ってチェックできた項目1つにつき1点。各コンポーネントで3点満点を目指す。

### 1. ナレッジクラリティ（最大3点）
- [ ] 各ページが、そのページの主要な問いに対する1文回答から始まる（Answer-first）
- [ ] ドメイン固有の用語は初出時に定義されている（未説明の専門用語なし）
- [ ] 各段落が単一のアイデアを含む（複数主張の段落なし）

### 2. 構造化フォーマット（最大3点）
- [ ] ページがセマンティックな H1 → H2 → H3 階層を使い、見出しレベルを飛ばしていない
- [ ] 意味のある全ページが JSON-LD を出力している（Article / TechArticle / FAQPage / Product / Organization 等）
- [ ] 比較コンテンツは散文ではなくテーブルを使う

### 3. 検索シグナル（最大3点）
- [ ] `/llms.txt` がサイトルートに存在し、主要ページを列挙している
- [ ] `/ai/` ディレクトリが各主要トピックのクリーンなMarkdownを提供（多言語サイトなら言語別にも）
- [ ] `robots.txt` が GPTBot, ClaudeBot, PerplexityBot, Google-Extended を明示的に許可。`sitemap.xml` が到達可能

### 4. 権威性シグナル（最大3点）
- [ ] 著者が検証可能な経歴を持ち、LinkedIn / GitHub / X / 出版プロフィールへの `sameAs` リンクがある
- [ ] 同じアイデンティティ（名前、役職、トピック）が少なくとも3プラットフォームで一貫している
- [ ] 著者が実際に発表した一次研究、書籍、論文へのリンクがサイト内にある

### 5. 引用シグナル（最大3点）
- [ ] 数値を使う主張は必ずソース名と年を引用している
- [ ] 各ページが `datePublished` と `dateModified` の両方を露出（JSON-LD または可視メタで）
- [ ] 比較コンテンツが業界標準（W3C, RFC, ISO, schema.org）を名前とリンクで参照

### 採点ガイド

| 合計 | 帯域 |
|-----|------|
| 13–15 | プロダクション級 — AIシステムに積極的に引用される |
| 9–12  | 良好 — AIから見えるが一貫性に欠ける |
| 5–8   | 部分的 — 検索性または権威性に大きなギャップ |
| 0–4   | 不可視 — `/llms.txt`、robots.txt、JSON-LD から開始 |

> もっと高いスコアが欲しい？ 各コンポーネントページ（ナレッジクラリティ、構造化フォーマット、検索シグナル、権威性シグナル、引用シグナル）に、スコアを 1 → 2 → 3 と動かす具体的実装が記載されている。
